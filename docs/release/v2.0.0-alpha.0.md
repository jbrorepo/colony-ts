The Colony is a local-first, security-first agent operating system built on Bun and TypeScript. Alpha 0 is a public source+Bun alpha centered on a real planner → worker → reviewer swarm path, conservative approvals, exact transcript truth, and Ollama-first provider setup.

## Quick start

```bash
git clone https://github.com/jbrorepo/colony-ts
cd colony-ts
bun install
bun run verify:alpha0         # 43-check launch gate (~30s)
bun run alpha0:provider-check # Ollama + optional cloud-key readiness
bun run start                 # launch the Ink terminal UI
```

Inside the UI:

```text
/doctor first-run
/swarm llm "draft a small local-first release note"
/swarm status <run_id>
```

Requires Bun ≥ 1.3 and (for the default local path) [Ollama](https://ollama.com) with a model pulled — e.g. `ollama pull llama3.2`.

## What ships in Alpha 0

- Local source+Bun install path
- Ollama-first provider stack with optional Anthropic / OpenAI-compatible / Gemini fallback (all four use raw `fetch()`, zero vendor SDKs)
- Real `AgentLoop` wired to the Ink terminal UI with anti-flicker streaming
- Conservative tool approvals (deny-by-default unless a session-scoped allow rule exists)
- Exact transcript truth + derived recall + MemPalace inspection foundations
- Real planner / worker / reviewer swarm with durable run snapshots, stage timeline, retry/resume/cancel
- Workflow DAG / checkpoint / approval / retry foundations
- Local-first GitHub issue/branch/worktree handoff planning behind approvals
- Local-only authenticated web-control operator shell
- MCP, skills, daemon, channel, and plugin marketplace lifecycle handoff surfaces
- 388-phase verification gate (`bun run verify:all`) + `tsc --noEmit` clean

## What is intentionally NOT shipped in Alpha 0

- Public hosted control plane
- Slack / Discord / Telegram delivery as a default alpha channel
- Credential persistence for external services
- Automatic push or PR creation
- Autonomous remote mutation without explicit approval
- Built-in live plugin registry fetch / install / activation UX

These are explicit boundaries, not omissions — see `docs/PROJECT_STATE.md` for the full list and `docs/release/ALPHA_0_RELEASE_READINESS.md` for the evidence behind each claim.

## Security posture

This release adheres to eight critical rules verified by the gate:

1. All security logic is pure TypeScript with zero npm dependencies
2. All LLM providers use raw `fetch()`
3. Default tool approval is conservative
4. No blocking I/O in the main runtime
5. Tool results over 10 KB are externalized and redacted before persistence
6. Canonical transcript truth is kept verbatim (modulo secret redaction) and separate from derived artifacts
7. Ink streaming is anti-flicker (ref-buffered, 50 ms flush cadence)
8. The log sanitizer installs before any module that may emit secrets

Report security issues via this repo's Security tab — see [`SECURITY.md`](https://github.com/jbrorepo/colony-ts/blob/main/SECURITY.md).

## Verification

CI runs the full gate (`verify:all` = 388 phases + `tsc --noEmit` + binary compile) on Ubuntu on every push and PR. The Alpha 0 commit was verified green on Linux as well as Windows.

## Known limitations

- Tested primarily on Windows; the Linux CI run confirms portability but day-to-day developer experience is Windows-first
- Two local reasoning-capable model paths are smoke-tested (`llama3.2`, `gemma4:latest`); other Ollama models will work but are not in the smoke matrix
- Compiled binary (`colony.exe` / `colony`) is produced by the build but not signed
- Interactive terminal UI requires a TTY — there is a headless smoke at `src/smoke-alpha0-headless.ts` for automation
