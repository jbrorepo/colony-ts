# Antelligence Launch Checklist

Use this checklist before launch materials, docs, website copy, demos, or sales-facing collateral reference Antelligence.

## Inclusion Criteria

- [ ] `ANTELLIGENCE_PLAYBOOK.md` is linked from `COLONY_BIBLE.md`.
- [ ] `ANTELLIGENCE_PLAYBOOK.md` is linked from `ROADMAP.md`.
- [ ] `ANTELLIGENCE_PLAYBOOK.md` is linked from `MARKET_GAP_ANALYSIS.md`.
- [ ] The Antelligence templates exist under `docs/templates/antelligence/`.
- [ ] Launch copy describes Antelligence as an operating model and playbook.
- [ ] Launch copy preserves Colony identity: local-first, security-first, caste/swarm model, MemPalace, conservative approvals, and exact transcript truth.

## Claim-Safety Review

- [ ] No copy says The Colony is a fully automated enterprise platform.
- [ ] No copy says Slack or Discord default live delivery, public hosting, credential persistence, or autonomous runtime setup is shipped; only verified approval-gated host-owned setup handoff/executor gates may be claimed.
- [ ] No copy says cloud, mobile, IDE, or enterprise admin surfaces are available unless source truth verifies them.
- [ ] No copy implies AI removes human accountability.
- [ ] No copy presents derived summaries as canonical transcript truth.
- [ ] No copy says Antelligence replaces the Colony Bible, security mesh, approval model, or caste model.

## Template Review

- [ ] Ant-inspired names appear in explanatory narrative.
- [ ] Enterprise-standard names appear in templates and gates.
- [ ] Every template has owner, evidence, decision, risk, and follow-up sections where relevant.
- [ ] Templates avoid unverified runtime automation claims.
- [ ] Templates can be used independently by a human team without The Colony runtime.

## Launch Readiness Review

- [ ] Antelligence is represented as the enterprise product-development expression of The Colony.
- [ ] Runtime status remains grounded in `ROADMAP.md`, `EXECUTION_PLAN.md`, `GAP_ANALYSIS.md`, and `BENCHMARK_BOARD.md`.
- [ ] Deferred capabilities are named as roadmap items, not shipped features.
- [ ] Release notes include a clear boundary between philosophy, templates, and runtime surfaces.
- [ ] A reviewer has searched docs for risky claims before launch.
